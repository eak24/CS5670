Design Decisions

We focused on writing clean, self-documenting code for ease of understanding and debugging. We did not implement any helper functions or classes, partially in service of that goal.

We chose to leverage as much of the scipy/numpy functionality that we could. This lightened our workload, made our code easier to read, and kept our code efficient as many of these numpy/scipy functions are compiled and vectorized at the machine-code level.

The Harris Corner Score was calculated using a Sobel filter to determine the x and y derivatives, then the structure tensor was constructed for each pixel using those values. The R score was found using an alpha score of 0.1 and the structure tensor values were blurred using a 5X5 Gaussian mask with a 0.5 sigma.

In the MOPS-Descriptor routine, we chose to generate the transformation matrix on the fly from four decomposed parts (translation, scale, rotation, translation’) instead of finding one “master matrix”. This proved very useful for debugging and testing, because we were able to check the correctness of each step of the transformation independently. If we were to continue this project, we would compute a single “master” transformation matrix to save ourselves the three superfluous matrix multiplication operations we do to compute each MOPS descriptor. q

Extra Credit

We chose two extra credit options. The first, and most trivial, was to implement a KD tree for finding best matches between images. We used scipy.spatial.KDTree, which had all of the functionality we were looking for (finding the keypoint or keypoints in d2 that were closest to a given keypoint in d1). Unfortunately, in our testing, the fact that KDTree was implemented mostly in python, and cdist in optimized C meant that using cdist to compute the distance matrix was faster overall.

The second extra credit option we implemented was a custom feature descriptor. Our descriptor combines the theory of MOPS and SIFT by computing the MOPS on the x and y derivatives of the image. However, unlike standard MOPS, we do not normalize the final intensities because the gradients should be invariant to additive illumination changes. It performed surprisingly well in certain cases, and as can be seen in the above comparison done on Ethan’s face (the second guy in the “Pictures of Us”).  It is, in practice, robust to small projective homographies (note the matching on his hair and collar).  See the column in AUC averages section for more detailed performance results.

Conclusion

We wrote a keypoint detector, two feature descriptors, and two matching schema that performed well on various rotation/translation matching datasets. We saw that no one combination of detector/descriptor/matcher worked the best in all situations. A good next step would be to characterize where these detectors/descriptors/matchers perform the best. Another next step could be to make the detector scale invariant.
